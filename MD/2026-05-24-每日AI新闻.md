### 2. Coding Agent 进入"装技能包"时代：Skills 生态爆发，85,000+ 可安装技能跨 18 种工具通用

- **日期：** 2026年5月24日
- **公司：** Anthropic / Vercel / 全行业
- **简介：** 2026 年 5 月，GitHub Trending 上同时冒出一批指向同一方向的项目：Skill 设计规范、Skill 市场、跨工具可移植配置、Agent 记忆系统。自 Anthropic 2025 年 12 月发布 Agent Skills 开放标准以来，16+ 主流 AI 工具（Claude Code、Cursor、Codex、Gemini CLI、GitHub Copilot、JetBrains Junie 等）已支持该标准。Vercel 推出的 skills.sh 平台索引了 85,000+ 个可安装 Skills，总安装量最高的 find-skills 已超 64 万次。Skills 的核心思想是：与其开发专用 Agent，不如把专业知识和工作流封装为通用 Agent 可即插即用的"技能包"——一行命令 `npx skills add` 即可安装。
- **关注点：** Skills 标准的爆发标志着 AI Coding 从"谁的 Agent 更聪明"转向"谁的生态更丰富"——这和移动互联网从"谁的手机更强"转向"谁的应用更多"的逻辑完全一致。Anthropic 通过制定 Skills 标准占据了"AI 的 App Store"生态位，而 Vercel 的 skills.sh 则充当了分发渠道。对开发者而言，一次编写、跨 18 种工具通用的 Skills 极大地降低了迁移成本和锁定风险。Skill 市场的成熟将加速 AI Coding 从"工具"向"平台"的跃迁。
- **举例：** 开发者在 Claude Code 中安装 `web-design-guidelines` Skill，Agent 自动获得生产级 Web 界面设计审查能力；安装 `azure-ai` Skill 后，Agent 可直接调用 Azure AI 服务；所有 Skill 在 Cursor、Codex、Gemini CLI 中同样可用，无需重复配置。

### 3. Raindrop Workshop 开源：本地 Agent 调试器 + 自修复评估循环，Claude Code 可直接读取修复

- **日期：** 2026年5月14日
- **公司：** Raindrop AI
- **简介：** AI Agent 可观测性公司 Raindrop 开源了本地调试器 Workshop（v0.1.6），这是首个完全本地运行的 AI Agent 调试工具。Workshop 作为本地守护进程，实时流式传输 Agent 的每一个 token、工具调用和决策到本地仪表盘（localhost:5899），所有追踪数据存储在单个轻量 SQLite 数据库中。核心亮点是"自修复评估循环"：Claude Code 或 Codex 可通过 MCP 协议读取 Workshop 的追踪记录，自动编写评估测试，检测故障，修复代码并重新运行，直至所有断言通过。Workshop 支持 TypeScript、Python、Rust、Go，兼容 Vercel AI SDK、OpenAI SDK、Anthropic SDK、LangChain、CrewAI 等主流框架。
- **关注点：** Workshop 解决的是 Agent 开发中最棘手的问题——"Agent 为什么做出了非预期的判断"。传统调试靠日志和 API 调用记录几乎无法回溯 Agent 的决策路径，Workshop 通过逐 token 流式追踪 + MCP 接入让 AI 自身参与调试闭环，实现了"Agent 帮你调试 Agent"的范式创新。这对 AI Coding 方向的意义在于：当 Claude Code 自身可以通过 Workshop 发现并修复 Agent 行为偏差，开发者的调试成本将从"人工逐条排查"降为"AI 自动修复+人工复核"。
- **举例：** 一个兽医助手 Agent 未询问必要的后续问题，Workshop 捕获完整轨迹后，Claude Code 读取该轨迹、编写特定评估、识别提示词中的逻辑缺陷、修复代码并自动重新运行——整个修复循环无需人工干预。

- **简介：** 在 Create 2026 百度 AI 开发者大会上，百度创始人李彦宏首次提出 AI 时代的核心度量衡——日活智能体数（DAA，Daily Active Agents），与移动互联网时代的 DAU 相对应。李彦宏认为，Token 只代表成本不代表收益，衡量投入而非产出；DAA 关注每天有多少智能体在替人完成任务闭环，更接近价值本质。他预测未来全球 DAA 有望突破 100 亿（当前全球 DAU 最高的 Meta 超过 34 亿）。李彦宏还提出"超级个体"概念：一个人加一支智能体编队，正取代传统的"两张披萨团队"成为最小生产力单元。
- **关注点：** DAA 的提出是对当前"Token 消耗竞赛"的有力反思——如果 10 亿 Token 的消耗只产生了 1 个完成任务的 Agent，那 1 亿 Token 产出 100 个 Agent 的平台显然更有价值。DAA 作为度量衡的优势在于：它衡量的是"交付结果"而非"消耗资源"，直接对应商业价值。对 AI Coding 方向而言，DAA 意味着平台竞争的焦点将从"谁支持更长的上下文"转向"谁的 Agent 每天能完成更多真实任务"——这与 Skills 生态的爆发方向一致：更多可复用技能 = 更多可交付的 Agent 任务。
- **举例：** 一个 Agent 平台日消耗 1000 亿 Token 但只完成了 10 万个任务（低 DAA），另一个日消耗 100 亿 Token 但完成了 100 万个任务（高 DAA）——DAA 更高的平台显然更健康；企业用 DAA 衡量内部 AI 平台的 ROI，而非简单的 Token 用量。

## 五、今日趋势总结

1. **AI Coding 从"造 Agent"走向"装技能包"：Skills 标准统一催生 AI 的 App Store 时代**
   Anthropic 的 Skills 开放标准在短短 5 个月内获得 16+ 主流工具支持、85,000+ 可安装技能，标志着 AI Coding 的竞争维度从"谁的 Agent 更聪明"转向"谁的生态更丰富"。ECC 的 156 项技能和 Skills 市场的爆发共同指向一个趋势：Agent 的通用能力已经足够强，差异化竞争的关键在于专业知识的封装和分发。这和移动互联网从"硬件参数比拼"转向"应用生态竞争"的逻辑完全一致——Skills 就是 AI 的 App Store。

2. **Agent 调试进入"自修复"时代：Workshop 让 AI 帮你调试 AI，闭环效率质的飞跃**
   Raindrop Workshop 的"自修复评估循环"开创了一个新范式：当 Agent 出现行为偏差，不再需要人工逐条排查日志，而是让 Claude Code 通过 MCP 读取追踪记录、自动编写评估、修复代码并重新运行。这与 ECC 的多 Agent 编排、Skills 的可复用技能包形成三层基础设施：编排层（ECC）负责分工，技能层（Skills）提供专业能力，调试层（Workshop）保证质量——AI Coding 的"操作系统"正在成形。

3. **前沿 AI 公司为"AI 训练 AI"做制度准备：RSI 安全岗位 + Karpathy 入 Anthropic 双线并进**
   OpenAI 招募 RSI 安全研究员（年薪 44.5 万美元）和 Karpathy 加入 Anthropic 主攻 RSI 预训练加速，两条线索共同表明：前沿 AI 公司正在为"AI 优化 AI"的递归自我改进时代做制度和技术双重准备。Anthropic 联合创始人 Jack Clark 估计 2028 年底前 AI 研发完全脱离人类参与的概率约 60%——这不再是科幻，而是需要立即应对的现实规划。

4. **AI 助手从"云端对话"走向"系统级操控"：Marvis 预示个人 AI 终端时代到来**
   腾讯 Marvis 的"操作系统层级"定位 + 6 Agent 协作体系，标志着 AI 助手正在从"聊天工具"升级为"系统级智能中枢"。与 Perplexity Personal Computer 的本地文件直控、微软 Copilot 的系统集成相比，Marvis 的差异化在于"出厂即预置"——用户无需配置即可使用。当 AI 助手能直接操控系统设置、文件管理、应用调用和跨端调度，个人 AI 终端的形态正在从概念走向产品。

5. **AI 行业度量衡之争：Token vs DAA，"消耗了多少"让位于"交付了多少"**
   百度李彦宏首提 DAA（日活智能体数）作为 AI 时代新度量衡，直指当前行业"Token 消耗竞赛"的荒谬——如果 10 亿 Token 的消耗只产出了 1 个完成任务的 Agent，那 1 亿 Token 产出 100 个 Agent 的平台显然更有价值。DAA 的提出与 Skills 生态的爆发形成呼应：更多可复用技能 = 更多可交付的 Agent 任务 = 更高的 DAA。AI 行业的价值评估正在从"算力消耗"转向"任务交付"。
