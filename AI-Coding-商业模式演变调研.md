# AI Coding 行业商业模式演变调研（2023—2026）

> 调研时间：2026 年 5 月 27 日 | 研究范围：全球 AI Coding 赛道核心玩家商业模式变迁

---

## 一、研究摘要

AI Coding 是大模型时代最先跑通 PMF（产品-市场契合）的垂直赛道。2023 年至今，行业商业模式经历了 **「订阅制确立 → Freemium 竞争 → 按量付费转型 → Agent 计价重构」** 四个阶段。每次转折的底层驱动力，都是技术范式的跃迁（补全 → 对话 → Agent → 云端 Agent），而定价模型被迫追赶技术现实。

**核心结论：**

1. **技术范式每升级一次，商业模式就重构一次**——补全时代支撑纯订阅制，Agent 时代倒逼按量付费。
2. **中间层公司的定价权正在被挤压**——模型商自建工具（Claude Code / Codex）、开源工具零加价（Cline），两头夹击订阅制中间商。
3. **ARR 增速惊人但盈利困境凸显**——Cursor ARR 从 0 到 10 亿美元仅用 2 年，但 2025 年中定价风波暴露出毛利为负的结构性问题。
4. **中国市场走「免费+生态」路线**——字节 Trae、阿里通义灵码均以免费切入，靠云服务和模型 API 变现，与海外订阅+按量付费路径明显分化。

## 二、商业模式演变的四个阶段

### 阶段一：订阅制确立期（2022 Q3 — 2023 Q4）

**技术范式：** Tab 补全 + 简单对话

| 事件 | 时间 | 意义 |
|------|------|------|
| GitHub Copilot 正式上线个人版，$10/月 | 2022.06 | 行业首个规模化 AI 编程订阅产品 |
| Copilot Business 发布，$19/用户/月 | 2022.11 | 首次分层定价，验证企业付费意愿 |
| Copilot ARR 突破 $1 亿 | 2023.10 | 证明 AI Coding 订阅制可盈利 |
| Cursor 由 Anysphere 推出 | 2023.01 | 以「AI 原生 IDE」切入，Pro 定价 $20/月 |
| Cursor 完成 OpenAI 领投 $800 万种子轮 | 2023.10 | 资本首次押注 Copilot 以外的 AI Coding 创业 |

**商业模式特征：**

- **纯订阅制（Flat-rate Subscription）**：按月/按年收取固定费用，用户享有无限或上限次数的使用权
- **Copilot 的 $10/月成为行业定价锚点**——后续产品基本围绕 $10—20 区间定价
- **核心逻辑**：补全任务的 Token 消耗低且可预测，固定订阅足以覆盖 API 成本

**技术范式：** 对话式编程 + 跨文件编辑

| 事件 | 时间 | 意义 |

- **竞争白热化**：Copilot 主动降价，Cursor ARR 狂飙，市场从一家独大走向多强并立

### 阶段三：按量付费转型期（2025 Q1 — 2025 Q3）

**技术范式：** Agent 模式 + 长上下文推理

| 事件 | 时间 | 意义 |
|------|------|------|
| Anysphere 完成 B 轮 $1.05 亿融资，估值 $26 亿 | 2025.01 | 资本加速涌入 |

| OpenAI 推出 Codex CLI（开源） | 2025.04 | 模型商直接下场做工具层 |
| GitHub Copilot 推出 Pro+，$39/月 | 2025.04 | 高级模型请求额外计费 $0.04/次 |
| OpenAI 拟 $30 亿收购 Windsurf | 2025.05 | 史上最大 AI Coding 收购案 |
| Cursor 定价风波：从按次计费改为 Token 计费 | 2025.06 | 行业首次大规模定价模式转型 |
| Cursor 推出 Ultra 方案，$200/月 | 2025.06 | 高端市场定价锚点上移 10 倍 |
| Cursor ARR 突破 $5 亿 | 2025.06 | 90 天翻倍，增长势能惊人 |

**商业模式特征：**

- **从按次计费（Request-based）到按 Token 计费（Usage-based）**：Agent 请求的 Token 消耗差异极大（简单补全 vs 复杂重构可达 10 倍差距），固定次数计费不可持续
- **Cursor 定价风波的教训**：$20/月 Pro 方案改为 Token 额度池制，部分用户月费暴涨至 $44 甚至 $1000+，暴露出中间层公司的毛利困境——API 成本随 Agent 使用量指数级增长，订阅收入无法覆盖
- **模型商自建工具层**：OpenAI 推出 Codex/Codex CLI，Anthropic 推出 Claude Code，直接面向终端用户，压缩中间商生存空间
- **开源+BYOK（Bring Your Own Key）模式崛起**：Cline、Aider 等开源工具零加价，用户直接向 API 供应商付费，透明度远超商业产品
- **高端市场打开**：Cursor Ultra $200/月、Devin $500/月、Claude Max $200/月，AI Coding 的 ARPU 显著高于传统 SaaS

### 阶段四：Agent 计价重构期（2025 Q4 — 2026 至今）

**技术范式：** 云端 Agent + 多 Agent 协作 + Spec 驱动开发

| 事件 | 时间 | 意义 |
|------|------|------|
| Cursor CEO 提出「AI 编程三时代论」 | 2026.03 | Tab 补全 → 本地 Agent → 云端 Agent 的范式划分 |
| Cursor 云端 Agent 已创建 35% 的代码提交 | 2026.03 | Agent 从辅助变为主力 |
| Devin 2.0 推出 ACU 计价单位 | 2025 下半年 | 「15 分钟 Agent 工作量」= 1 ACU，抽象化计价 |
| OpenAI-Windsurf 交易破裂 | 2025.07 | 微软 IP 条款阻拦 |
| 谷歌 $24 亿吸纳 Windsurf 核心团队 | 2025.07 | 人才争夺替代公司收购 |
| Cognition 以 $3 亿收购 Windsurf 剩余资产 | 2025.07 | 客户+IP+品牌整合 |
| Anysphere 完成 D 轮 $23 亿融资，估值 $293 亿 | 2025.11 | 全球 AI Coding 估值之最 |
| Cursor ARR 突破 $10 亿 | 2025.11 | 史上增长最快的 SaaS 之一 |
| Cursor 推出自研模型 Composer + Tab | 2025.10—11 | 降低对外部模型依赖，提升毛利 |
| Claude Code 活跃用户增长 300%，收入增长 5.5 倍 | 2025.05 后 | Claude 4 模型推动 |
| Trae 全球用户超 600 万，月活 160 万 | 2025 年底 | 中国市场 AI Coding 最大用户基数 |

**商业模式特征：**

- **抽象化计价单位**：Devin 的 ACU（Agent Compute Unit）将计算资源、推理、网络打包为「15 分钟工作量」，用户无需理解 Token 细节
- **云端 Agent 计费新范式**：Agent 运行在独立 VM 上，计费从「输入/输出 Token」转向「任务完成度+计算资源」
- **自研模型成为毛利关键**：Cursor 推出 Composer/Tab，减少对外部模型 API 的依赖，从纯渠道商向模型+产品双驱动转型
- **国内走免费生态路线**：字节 Trae、阿里通义灵码免费使用，靠模型 API 和云服务变现
- **多形态并行**：IDE + CLI + Cloud 三种产品形态并存，用户按场景选择，计费方式各自适配

## 三、五大关键转折点

### 转折 1：Copilot 订阅制验证（2023 Q4）

**标志性事件：** Copilot ARR 突破 $1 亿并盈利

**驱动力：**
- 大模型代码生成能力（GPT-3.5/4）首次达到商用水平
- GitHub 生态垄断：1.5 亿开发者 + VS Code 市占率
- 补全场景 Token 消耗低，$10/月即可覆盖成本+盈利

**对行业的影响：** 确立了「AI Coding = 订阅制 SaaS」的基本假设，成为后续所有创业公司的定价参照系。

---

### 转折 2：Cursor 打破 Copilot 垄断（2024 Q2—Q4）

**标志性事件：** Cursor ARR 从 $0 到 $6500 万，同比增速 6400%

**驱动力：**
- **产品范式创新**：从「IDE 插件」升级为「AI 原生 IDE」——Cursor 不只是补全，而是重构了开发者与 AI 的交互方式
- **Composer 跨文件编辑**：突破了 Copilot 的单文件补全限制
- **Andrej Karpathy 等意见领袖背书**，社区口碑爆发

**对行业的影响：** 证明 Copilot 并非不可挑战；AI Coding 的竞争焦点从「谁的补全更准」转向「谁的交互更智能」。也催生了「AI Coding 可以跑出超级独角兽」的资本预期。

---

### 转折 3：Agent 模式倒逼定价重构（2025 Q2—Q3）

**标志性事件：** Cursor 2025 年 6 月定价风波

**驱动力：**
- **Agent 请求的 Token 消耗不可预测**：一个简单补全 100 tokens，一个完整 PR 可能耗 50,000+ tokens——按次计费导致重度用户「薅羊毛」，公司毛利为负
- **上下文窗口扩大**：Agent 需要理解整个代码库，输入 Token 成本指数级增长
- **外部模型 API 成本刚性**：Cursor 使用 OpenAI/Anthropic/Google 模型，成本无法压缩

**对行业的影响：** 全行业从 Flat-rate 转向 Usage-based 是不可逆趋势。Cursor 事件后，Augment Code、Zed 等纷纷跟进按量计费。但这也暴露了中间层公司的结构性困境：**你只是一个 API 的转售商，为什么用户要为你加价？**

---

### 转折 4：模型商下场做工具（2025 Q1—至今）

**标志性事件：** Claude Code / OpenAI Codex 直接面向终端用户

**驱动力：**
- **模型能力差异化缩小**：GPT、Claude、Gemini 编程能力趋同，模型商需要通过应用层锁定用户
- **Copilot 的前车之鉴**：OpenAI 提供 Codex 模型给 GitHub，但最赚钱的是微软，OpenAI 只收 API 费
- **自建工具 = 数据飞轮**：直接触达开发者，获取使用数据，反哺模型训练

**对行业的影响：** 中间层公司面临生存危机——模型商拥有定价权（可以零利润卖工具），开源工具拥有透明度优势，订阅制中间商两头受压。Cursor 的应对是自研模型，但这需要巨额研发投入。

---

### 转折 5：并购重组重塑竞争格局（2025 Q2—Q3）

**标志性事件：** Windsurf 收购案的三重转折

**驱动力：**
- **AI Coding 是模型商的「必争之地」**：编程是唯一 AI 产出能直接改进 AI 自身的领域（Karpathy 的「自我加速飞轮」论）
- **OpenAI 缺少自有 IDE**：Codex 模型通过 GitHub Copilot 变现，但控制权在微软
- **微软 IP 条款约束**：微软有权访问 OpenAI 的知识产权，导致 OpenAI 不愿让 Windsurf 技术流入微软

**对行业的影响：**
- **人才 > 资产 > 公司**：谷歌花 $24 亿只雇走核心团队+非独家授权，不买公司——人才是最稀缺资源
- **垂直整合加速**：模型商必须控制工具层，工具商必须自研模型——中间态越来越难存活
- **定价权向上游集中**：模型商掌握 Token 定价权，工具层只能被动接受成本波动

## 五、驱动因素深度分析

### 5.1 技术驱动力

```
补全（Tab）→ 对话（Chat）→ Agent（本地）→ 云端 Agent（自主）
   ↓              ↓              ↓                ↓
 低 Token 消耗   中等消耗       高消耗+不可预测   计算资源密集
   ↓              ↓              ↓                ↓
 纯订阅可行     订阅+限次      按量付费必然      抽象化计价
```

**关键洞察：** 每次技术范式升级，都导致单用户服务成本的增长速度远超付费意愿的增长速度。订阅制的「无限使用」承诺，在 Agent 时代变得不可持续——因为一个重度 Agent 用户消耗的成本，可能是一个轻度补全用户的 50 倍以上。

### 5.2 竞争驱动力

- **模型商垂直整合**：OpenAI（Codex → Windsurf 尝试）、Anthropic（Claude Code）、Google（Jules + Gemini）——模型商不再满足于卖 API，要直接锁定开发者
- **开源生态侵蚀定价权**：Cline 9 个月 300 万安装量，证明零加价模式可以快速获客——当开发者可以直接用 API 价格使用同样的模型，为什么多付 20% 加价？
- **中国市场免费战**：Trae/通义灵码完全免费，将 AI Coding 视为云服务和模型 API 的入口而非利润中心

### 5.3 成本驱动力

- **推理成本虽在下降，但 Agent 消耗量指数增长**：单 Token 成本 2 年降 10 倍，但 Agent 任务消耗的 Token 量增长了 100 倍
- **多模型路由成本**：Cursor 同时调用 OpenAI/Anthropic/Google/自研模型，成本结构复杂
- **自研模型投入**：Composer/Tab 研发需要巨额 GPU 训练成本，但可降低长期 API 依赖

### 5.4 资本驱动力

- **估值泡沫与增长压力**：Cursor 估值 2 年从 $0 到 $293 亿，资本期望持续超高速增长
- **并购溢价**：Windsurf $30 亿报价（ARR ~$1 亿，30x 倍率），说明资本对 AI Coding 赛道的长期信心
- **模型商融资支撑工具层投入**：OpenAI $400 亿融资、Anthropic 持续融资，使模型商有能力零利润甚至负利润运营工具层

## 六、趋势判断与未来展望

### 6.1 近期趋势（2026）

1. **按量付费全面取代纯订阅制**——所有 Agent 类产品都将转向 Token/ACU 计费
2. **自研模型成为中间层公司的生存前提**——Cursor 的 Composer/Tab 是标杆，Windsurf/Codeium 也将跟进
3. **MCP 生态繁荣**——模型上下文协议（MCP）推动工具互操作，开源生态进一步壮大
4. **Spec 驱动开发成为新范式**——从「人写代码 AI 补全」到「人写 Spec AI 写代码」，开发者角色从编码者变为编排者

### 6.2 中期趋势（2027—2028）

1. **云端 Agent 成为主流**——Cursor 数据显示 35% 代码提交已由云端 Agent 完成，比例将持续提升
2. **多 Agent 协作系统**——专业化的测试 Agent、安全 Agent、部署 Agent 协同工作
3. **行业垂直化**——安全审计、数据库优化、云部署等场景出现专用 Agent
4. **定价锚点从「人月」转向「Agent 工时」**——Devin 的 ACU 模式可能成为行业标准

### 6.3 结构性风险

1. **中间层公司的护城河不够深**——当模型商和开源工具两头挤压，纯工具层公司的估值倍数可能回调
2. **Token 成本波动性**——模型商掌握定价权，工具层公司无法控制核心成本
3. **代码质量与安全风险**——AI 生成代码 35%+，但幻觉、风格不一致、安全漏洞等问题尚未解决
4. **开发者技能退化**——过度依赖 Agent 可能导致工程师的底层能力退化，形成「AI 依赖陷阱」

---

## 七、对 QQ 浏览器文件业务的启示

1. **AI 能力的商业化不必走订阅制**——中国市场用户对订阅付费接受度低，免费+增值/生态变现更可行
2. **Agent 模式是下一个机会窗口**——Excel 助手、简历助手等场景天然适合 Agent 化，用户描述需求 → AI 端到端交付
3. **自研模型或深度定制是毛利关键**——通用大模型 API 成本高且不可控，场景化小模型+蒸馏是可行路径
4. **Freemium 是获客最优解**——基础功能免费吸引用户，高级 Agent 能力按量付费
